　　针对以上这个问题，可以每次删除后，将整个队列中元素向前移动一位，也就是front被固定，队尾rear=rear-1,但是这么做，如果队列大的话，每次移动的元素就太多，性能会大幅下降。于是我们就有另一个比较好的办法：

循环队列

  4. 遍历时，要根据front和rear的大小进行遍历，若rear大,直接遍历一部分即可，如果front>rear,则分front-capacity和0-rear两部分遍历。
